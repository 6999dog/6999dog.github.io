重新捡起c语言（一）

2020年04月19日 822 字 大概 3 分钟

昨天晚上的时候准备写ACM试题，真的绝了
看着题目分析了一下思路，马上就能想到该怎么写
但是我实际写的时候由于一些晓得细节和语法连连报错
从今天开始把C语言捡起来

首先就来谈谈我昨天遇到的问题吧

  1. 在main函数的末尾，我就没有写return 0;
    后来我就去查了一下它的作用
    return 0; 语句用于表示退出程序
  2. 昨天在写字符串的判断的时候直接就写了字符串并没有声明
    char desc = “desc”
    这样之后才没有报错
  3. 在使用指针数组的时候没有开辟空间
    对头文件的使用也王的一干二净

    科普一下：stdio.h 是一个头文件 (标准输入输出头文件) ,
    #include 是一个预处理命令，用来引入头文件。
    当编译器遇到 printf() 函数时，如果没有找到 stdio.h 头文件，
    会发生编译错误
    由此可见对于头文件的引用会影响之后的语句

  4. 最近写多了网页js之类的语句
    习惯了用缩进来控制函数结构，
    很多语句都掉了{}
  5. 发现自己写代码的时候没有注释的习惯，很多时候自己都看不懂自己的代码
    这一点必须马上改过来

正式开始我捡起C语言的行动了

  1. 字符和字符串的表示
    *’a’单引号里面只能用一个字符
    *”snai”双引号里面可以引用多个字符

  2. 变量的声明
    *extern表示这个变量在其他的地方应用，不需要存储空间
    *例如：
    *#include <stdio.h>

    /外部变量声明/
    extern int x ;
    extern int y ;
    int addtwonum()
    {
    return x+y;
    }

  3. 定义常量
    *使用#define在头文件下面声明
    *使用const关键字在数据前面声明 const int x =10;

  4. 运算符
    刚刚在运算符里面看到了，我一直以为是单纯的指针
    **运算符的作用就是指向一个变量
    *后续在指针里面继续思考

  5. 循环
    *其实循环这个没有什么好说的
    *但是我觉得最重要的就是自己的思维要清晰
    *控制好循环的次数，不要陷入死循环

  6. 函数
    *这一块带给我的疑惑就是传值的方式问题
    *我绝大多数的时候都是直接使用了传值调用的方法
    *我不会用引用方式传入函数，这里放个例子吧
    *后续的过程中我会一直加油练习编程，用实操来锻炼自己

    #include <stdio.h>
    /* 函数声明 /
    void swap(int *x, int *y);
    int main ()
    {
    /     局部变量定义 */
    int a = 100;
    int b = 200;
    printf(“交换前，a 的值： %d\n”, a );
    printf(“交换前，b 的值： %d\n”, b );
    / * 调用函数来交换值

    &a 表示指向 a 的指针，即变量 a 的地址
&b 表示指向 b 的指针，即变量 b 的地址
    • /
      swap(&a, &b);
      printf(“交换后，a 的值： %d\n”, a );
      printf(“交换后，b 的值： %d\n”, b );
      return 0;
      }

  7. 数组
    *这里其实还是指针的事，通过指针来传递数组
    *还有熟悉如何定义数组
